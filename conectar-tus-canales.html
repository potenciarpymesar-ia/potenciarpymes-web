Conectar Tiendanube, Mercado Libre e Instagram

Cómo dejar de actualizar stock y precios a mano en 3 lugares distintos, sin contratar un desarrollo a medida.

Esto no es un plugin que instalás y ya — es la arquitectura que tenés que entender antes de conectar nada. Una vez que la tenés clara, el resto es configuración.

El problema real

Vender en Tiendanube, Mercado Libre e Instagram al mismo tiempo significa que el mismo dato vive en 3 lugares: stock, precio, y a veces hasta la descripción del producto. Si actualizás uno a mano y te olvidás de los otros dos, vendés algo que no tenés, o dejás plata en la mesa con un precio viejo.

La arquitectura simple

La idea central: elegí una fuente de verdad (normalmente Tiendanube, porque tiene el catálogo más completo) y todo lo demás se actualiza automáticamente a partir de ahí. No al revés, y no los tres editándose entre sí — eso es lo que genera los quilombos.

1. Tiendanube manda

Ahí cargás producto, precio, stock y fotos una sola vez. Es tu catálogo maestro.

2. Un webhook avisa cuando algo cambia

Tiendanube tiene una API de webhooks real (no hace falta que un desarrollador te la programe desde cero): te suscribís al evento product/updated, que se dispara cada vez que cambia un producto — incluido el stock — y también hay eventos para pedidos (order/created, order/paid).

3. n8n recibe el aviso y replica el cambio

Un workflow simple escucha ese webhook, y usa la API de Mercado Libre para actualizar el mismo producto allá. Instagram Shopping generalmente se sincroniza solo si usás el catálogo de Meta conectado a Tiendanube — ahí no hace falta este paso extra.

El registro del webhook (esto es real, no pseudocódigo)

Así se suscribe tu app al evento de Tiendanube — esto es lo que hay que llamar una sola vez para que empiece a avisarte:

POST https://api.tiendanube.com/v1/{store_id}/webhooks 
{
  "event": "product/updated",
  "url": "https://tu-n8n.com/webhook/producto-actualizado"
}

Y así se ve la llamada real a Mercado Libre para actualizar el stock de ese mismo producto:

PUT https://api.mercadolibre.com/items/{item_id} 
{
  "available_quantity": 6
}

El workflow completo en n8n

Flujo en n8n (esqueleto) 
Webhook (recibe el evento product/updated)
  → Code: extraer product_id y nuevo stock
  → HTTP Request (PUT api.mercadolibre.com/items/{id})
      body: { "available_quantity": {{ $json.stock }} }
  → IF (respuesta OK)
      → Log en Google Sheets (auditoría de cambios)
      → ELSE: Notificación por email/Slack (falló la sincronización)
Ojo con esto: necesitás las credenciales de API de Mercado Libre (App ID + Client Secret, se generan gratis en su portal de desarrolladores) y el ID de cada producto mapeado entre las dos plataformas. Ese mapeo inicial es manual una sola vez — después queda automático. Si tu cuenta de Mercado Libre usa stock multiorigen (varios depósitos), no se actualiza así — se maneja con los endpoints de User Products, que son distintos.

Qué NO conviene automatizar todavía

Si recién estás empezando con esto, no automatices precio (los ajustás poco) ni descripciones (cambian menos). Empezá solo por stock — es lo que más rápido te genera problemas si se desincroniza, y es la automatización con el ROI más claro.

Fuentes técnicas verificadas: documentación oficial de Webhooks de Tiendanube y documentación oficial de stock en la API de Mercado Libre.
